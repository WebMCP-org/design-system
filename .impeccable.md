# Impeccable

## Design Context

### Users

Developers building and operating agentic web apps (SigVelo, Nanites, WebMCP). They consume this design system as `@mcp-b/react-components` + `@mcp-b/design-tokens` while building product UI — dashboards, chat/agent surfaces, forms, and tool-call views. Context is dense, work-focused screens where accuracy and scannability beat spectacle.

### Brand Personality

Technical, calm, trustworthy. The interface should evoke confidence and quiet competence — an instrument panel, not a landing page. Agent activity (tool calls, reasoning, streaming) should read as legible and inspectable, never magical or noisy.

### Aesthetic Direction

Product UI: minimal and borderless-first, neutral-forward with restrained primary accent, equal-quality light and dark modes (semantic `--sigvelo-*` fill/text/stroke tokens, oklab-derived palettes). Containment comes from whitespace and quiet surfaces, not chrome: agent-activity collapsibles (Tool, Task, Reasoning) are Ploy-style hairline-outlined boxes on the canvas with muted header rows and plain content; passive blocks (CodeBlock, Terminal) are borderless fills; elevated surfaces (Card, Artifact, overlays) are shadow-led; interactive fields (Input, PromptInput) keep real borders. Expandable panels are height-bounded (20rem) with internal scroll so they never displace the conversation scroll. Agent activity nests two collapse levels max (Activity → row → quiet fill panel like the "Shell" block, never a chromed component like Terminal inside the outlined box); collapsed headers are derived natural-language summaries ("Thought, ran 2 commands") and rows are verb+command, no tool names. Radii 3–16px (containers 12px+), fast subtle motion (150–360ms, standard/emphasized easings, reduced-motion collapses to 0ms). Anti-references: cards-in-cards chrome, marketing gradients, glassmorphism, playful/bouncy motion, Tailwind-style utility restyling of component internals.

### Design Principles

1. Tokens are the only source of color, space, type, radius, and motion — never hardcode values; never redeclare `--sigvelo-*` in components.
2. Base UI owns behavior and state (data attributes are the styling contract); CSS adds only MCP-B visuals not already in `styles/base.css`.
3. Smallest tasteful change wins — preserve public APIs, props, and tests; polish is subtractive and corrective before it is additive.
4. Accessibility is non-negotiable: visible focus (canonical ring: `box-shadow: 0 0 0 2px canvas, 0 0 0 4px focus`; fields soften the outer layer to a 80%-transparent halo plus a focus-color border), forced-colors and reduced-motion support, semantic HTML, WCAG AA contrast in both themes.
5. Density with hierarchy — product screens are compact, but weight, spacing rhythm, and color contrast must make structure obvious at a glance.
